# DCVC-RT 神经编解码实验

> 对应设计文档: `docs/superpowers/specs/2026-07-11-dcvc-rt-experiment-design.md`

## 使用步骤

1. 确认 DCVC 仓库已 clone 并编译（见 Task 1）
2. 准备帧序列: `uv run python experiments/dcvc-rt/prepare_frames.py`
3. 跑 DCVC-RT: `uv run python experiments/dcvc-rt/run_dcvc.py`
4. 跑 H.265 对照: `uv run python experiments/dcvc-rt/compare_h265.py`
5. 查看结果: `cat experiments/dcvc-rt/results/comparison.json`

## 结论

### 实验环境

- GPU: NVIDIA RTX 4090
- 测试序列: `C104_20260115121711_10s_640x480_6fps.mp4`（10 秒，640x480，60 帧）
- DCVC-RT QP: I=4, P=4

### 编码性能

| 指标 | DCVC-RT |
|------|---------|
| P 帧编码速度 | ~438 fps |
| P 帧解码速度 | ~419 fps |
| I 帧编码速度 | ~0.3 fps（瓶颈） |
| 端到端整体 fps | 7.41 fps |

P 帧编解码性能极高（>400 fps），但 I 帧编码耗时约 7.8 秒（占总时间 96%），严重拖累整体吞吐。对于实时视频传输场景，I 帧瓶颈是关键障碍；对于离线批处理场景影响较小。

### 码率-质量对比（DCVC-RT vs H.265）

| 指标 | DCVC-RT (3.4 kbps) | H.265 (7.0 kbps) | 差异 |
|------|---------------------|-------------------|------|
| PSNR-Y | 28.49 dB | 24.76 dB | +3.73 dB |
| SSIM | 0.697 | 0.613 | +0.084 |
| LPIPS | 0.594 | 0.601 | -0.007（略优） |
| 码率 | 3.4 kbps | 7.0 kbps | DCVC-RT 仅为 H.265 的 49% |

**核心结论**: DCVC-RT 在 **一半码率** 下实现了 **+3.7 dB** 的 PSNR 提升，码率-质量效率显著优于 H.265。SSIM 也有明显优势，LPIPS 两者接近。

### 客观质量评估

- PSNR-Y 28.49 dB 处于"可接受"范围（>25 dB），但距离"较好"（>30 dB）仍有差距
- SSIM 0.697 表示帧间结构相似度一般，还原帧存在可见失真
- LPIPS 0.594 偏高（越低越好），说明感知质量仍有较大提升空间
- DCVC-RT 内部报告的 PSNR-RGB 为 28.30 dB，与 Y 通道 PSNR 一致

### 大运动稳定性

由于测试序列运动幅度有限（6 fps 低帧率静态场景为主），本次实验未充分验证大运动场景下的稳定性。需要补充高帧率、大运动序列（如机器人遥操作场景）进一步测试。

### 对语义传输项目的适用性评估

**优势:**
- 极低码率（3.4 kbps）下仍保持可接受的 PSNR，与语义传输"极低码率"目标契合
- P 帧编解码速度极快，适合连续帧传输
- 神经编解码在低码率区间的质量优势明显

**局限:**
- I 帧编码瓶颈导致整体 fps 仅 7.41，无法满足实时传输需求（需 >30 fps）
- LPIPS/SSIM 表明感知质量一般，还原帧存在神经编解码特有的失真
- 模型体积和显存需求尚未评估，部署可行性待确认
- 与现有 Diffusers 文生图接收端的整合路径不明确

### 建议

**推荐继续探索，但需解决 I 帧瓶颈后再评估实时可行性。** 具体建议:

1. **I 帧优化**: 探索 I 帧编码加速（模型量化、TensorRT 编译、或使用轻量 I 帧编码器）
2. **补充测试**: 使用机器人遥操作典型场景（高帧率、大运动）重新评估
3. **感知质量**: 对比 DCVC-RT 还原帧与 Diffusers 生成帧的主观质量
4. **部署评估**: 测量模型显存占用、加载时间，评估嵌入式/边缘部署可行性
