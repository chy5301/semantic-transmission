## 通用对比维度

### 视觉理解模型（发送端）

1. **描述精度**：能否准确捕捉场景中的物体、布局、颜色、纹理？
2. **结构化输出**：能否按指定模板生成分层描述（全局→局部→细节）？
3. **视频理解**：是否支持视频输入？能否捕捉时序变化？
4. **推理效率**：单帧处理延迟和吞吐量。
5. **部署灵活性**：本地部署 vs API 调用，显存需求。

### 生成模型（接收端）

1. **生成质量**：FID、CLIP Score、人工评估。
2. **条件控制**：支持哪些 ControlNet 条件？多条件融合效果如何？
3. **推理速度**：生成单帧的耗时，是否支持少步采样（如 Turbo/Lightning）。
4. **时序一致性**：视频生成时帧间是否连贯（仅视频模型）。
5. **ComfyUI 集成**：是否有现成的 ComfyUI 节点支持。

### 条件控制（ControlNet）

1. **信息保留**：该条件类型保留了原图的哪些信息（结构/深度/语义/纹理）？
2. **数据量**：条件图的典型大小，压缩后的传输成本。
3. **还原质量**：配合文本描述时的还原精度。
4. **提取速度**：从原图提取条件的计算成本。
5. **多条件融合**：与其他条件类型组合使用时的效果。
