## 1. Qwen2.5-VL 系列

### 基本信息

- **开发者 / 机构**：阿里巴巴 Qwen 团队
- **发布时间**：2025 年 1 月（技术报告 2025 年 2-3 月）
- **许可证**：Apache 2.0（开源，商用友好）

- **技术报告**：[arXiv:2502.13923](https://arxiv.org/abs/2502.13923)

### 参数量

| 版本 | 参数量 | 说明 |
|------|--------|------|
| Qwen2.5-VL-3B | ~3.7B | 轻量版，适合边缘部署 |

| Qwen2.5-VL-72B | ~73.4B | 旗舰版，GSC 论文使用此版本 |

### 能力概述

Qwen2.5-VL 是目前开源多模态模型中的旗舰之一。核心架构创新包括：
- **Window Attention ViT**：视觉编码器使用窗口注意力优化推理效率，支持 2D-RoPE
- **动态分辨率**：原生支持任意分辨率图像输入，无需固定裁剪
- **动态 FPS 视频采样**：将动态分辨率扩展到时间维度，支持小时级长视频
- **绝对时间编码**：MRoPE 对齐绝对时间，支持秒级事件定位
- **结构化输出**：支持 bounding box / point 定位，表格、发票、表单等结构化数据提取

72B 版本在文档理解、图表分析、长视频 QA 等任务上达到 GPT-4o 同级水平。

### 能力概述

InternVL2.5 是首个在 MMMU 基准上突破 70% 的开源多模态模型。核心优势：
- **大视觉编码器**：6B 参数 InternViT 提供更强的视觉特征提取
- **动态分辨率**：训练 max 36 tiles (448x448)，测试可扩展到 128 tiles
- **视频理解**：采样 32 帧，448 像素分辨率进行评估
- **多尺寸覆盖**：从 1B 到 78B 完整覆盖，适应不同部署场景

## 3. LLaVA-OneVision

### 基本信息

- **开发者 / 机构**：LMMs-Lab（ByteDance、NTU 等联合团队）
- **发布时间**：2024 年 8 月（LLaVA-OV）；2025 年 9 月（LLaVA-OV-1.5）
- **许可证**：Apache 2.0
- **HuggingFace**：[lmms-lab/llava-onevision-qwen2-7b-ov](https://huggingface.co/lmms-lab/llava-onevision-qwen2-7b-ov)
- **GitHub**：[LLaVA-VL/LLaVA-NeXT](https://github.com/LLaVA-VL/LLaVA-NeXT)
- **论文**：[arXiv:2408.03326](https://arxiv.org/abs/2408.03326)

### 参数量

| 版本 | 参数量 | 语言模型基座 |
|------|--------|------------|
| LLaVA-OV-0.5B | ~0.5B | Qwen2-0.5B |
| LLaVA-OV-7B | ~8B | Qwen2-7B |
| LLaVA-OV-72B | ~73B | Qwen2-72B |
| LLaVA-OV-1.5-4B | ~4B | Qwen3-4B（新版） |
| LLaVA-OV-1.5-8B | ~8B | Qwen3-8B（新版） |

### 能力概述

LLaVA-OneVision 是首个在单一模型架构中同时推进单图像、多图像和视频三个视觉场景性能边界的开源模型。特点：
- **统一架构**：单图/多图/视频三种模态共享同一架构
- **AnyRes 模式**：单图像最多 729*(9+1) tokens，多图每张 729 tokens，视频每帧 196 tokens
- **开源透明**：训练代码、数据集、模型权重全部公开
- **LLaVA-OV-1.5**：使用 Qwen3 作为新基座，在 27 个基准中全面超越 Qwen2.5-VL 同级模型

### 性能指标（LLaVA-OV-7B）

| 指标 | 数值 | 基准测试 | 说明 |
|------|------|----------|------|
| 48.8 | MMMU val | 多学科推理 | 弱于 Qwen2.5-VL-7B |
| 80.8 | MMBench | 通用视觉 QA | — |
| 61.7 | MMStar | 推理与识别 | — |
| 90.2 | DocVQA | 文档理解 | — |
| 63.2 | MathVista | 数学推理 | — |
| 96.0 | ScienceQA | 科学问答 | 强项 |

> **LLaVA-OV-1.5-8B 更新成绩**：在 27 个基准中 18 项超越 Qwen2.5-VL-7B，平均准确率 76.0% vs 74.2%。全部训练成本控制在 $16,000 内（128x A800 GPU，3.7 天）。

### 部署要求

| 项目 | LLaVA-OV-7B | LLaVA-OV-72B |
|------|-------------|--------------|
| 显存 (BF16) | ~16GB | ~150GB |
| 推荐 GPU | RTX 4090 / A100 | 2-4x A100 80GB |
| 推理框架 | Transformers / SGLang | SGLang |
| 量化支持 | GPTQ / AWQ | GPTQ / AWQ |

### 与本项目的适配性

**优势：**
- Apache 2.0 许可证，完全开源
- 训练过程完全透明可复现，便于针对语义传输场景微调
- 视频处理 token 效率高（每帧仅 196 tokens）
- LLaVA-OV-1.5 版本性能大幅提升，接近或超越 Qwen2.5-VL 同级

**劣势：**
- MMMU 分数 (48.8) 明显低于 Qwen2.5-VL-7B (58.6)，复杂场景理解较弱
- 视频理解的 benchmark 数据不如 Qwen2.5-VL 和 InternVL2.5 完整
- 原始版本（非 1.5）缺乏长视频支持和时间定位能力

**推荐：** 作为参考和微调基线，但不推荐作为首选。如果 LLaVA-OV-1.5 正式发布，值得重新评估。

## 4. GPT-4o / GPT-4V

### 基本信息

- **开发者 / 机构**：OpenAI
- **发布时间**：GPT-4V 2023 年 9 月；GPT-4o 2024 年 5 月
- **许可证**：商业 API，不开源
- **API 文档**：[OpenAI API](https://platform.openai.com/docs)

### 参数量

未公开。估计在数千亿参数级别。

### 能力概述

GPT-4o 是 OpenAI 的多模态旗舰模型，支持文本、图像、音频、视频输入输出。在语义理解任务上表现出色，但在几何任务（深度、法线估计）上弱于专用 CV 模型。

关键特性：
- 支持结构化输出（JSON mode / function calling）
- 1M token 上下文窗口（GPT-4.1 版本）
- 多模态原生融合

### 性能指标

| 指标 | 数值 | 基准测试 | 说明 |
|------|------|----------|------|
| ~70.3 | MMMU val | 多学科推理 | 与 Qwen2.5-VL-72B 相当 |
| ~83 | MMBench EN | 通用视觉 QA | 低于开源旗舰 |
| 71.9 | Video-MME (w/o sub) | 视频理解 | — |
| 77.2 | Video-MME (w/ sub) | 视频理解（含字幕） | — |

> 注：GPT-4.1（2025 年 4 月）在 Video-MME 上达到 SOTA。

### API 定价

| 模型 | 输入 ($/1M tokens) | 输出 ($/1M tokens) | 说明 |
|------|-------------------|-------------------|------|
| GPT-4o | $2.50 | $10.00 | 主力模型 |
| GPT-4o-mini | $0.15 | $0.60 | 轻量版 |
| GPT-4.1 | — | — | 最新版，1M 上下文 |

> 图像输入按 token 数计费，复杂图像 (1024x1024) 约几万 token，约 $0.01-0.05/张。
> 缓存输入半价 ($1.25/1M)，Batch API 半价。

### 与本项目的适配性

**优势：**
- 描述生成质量高，语义理解能力强
- 原生支持结构化输出（JSON mode），可直接生成规范化场景描述
- 无需本地 GPU，即开即用
- 可作为描述质量的 ground truth 基线

**劣势：**
- 商业 API，高频调用成本高（视频帧按帧调用成本极高）
- 网络延迟，不适合实时传输场景
- 数据隐私风险（视频内容上传到第三方）
- 不可本地部署，无法离线使用
- API 行为可能随版本更新变化

**推荐：** 仅作为质量评估基线，不适合作为生产部署方案。

## 语义传输场景适用性综合分析

### 发送端核心需求

语义传输发送端需要模型具备以下能力：
1. **精确场景描述**：准确捕捉物体、布局、颜色、纹理、空间关系
2. **结构化输出**：按指定模板生成分层描述（全局场景 → 区域 → 物体细节）
3. **视频时序理解**：捕捉帧间变化，识别运动和场景转换
4. **推理效率**：满足实时或近实时处理需求
5. **部署可行性**：在可用硬件上部署

### 模型推荐排序

#### 首选：Qwen2.5-VL-7B

- **理由**：GSC 论文验证了 Qwen2.5-VL 系列在语义传输中的有效性；7B 版本在单卡 RTX 4090 上可运行；支持动态分辨率和长视频；结构化输出能力强；Apache 2.0 许可证
- **适用场景**：原型开发、单卡部署、实时处理

#### 备选 1：InternVL2.5-8B

- **理由**：MVBench 和 MMBench 分数优于 Qwen2.5-VL-7B；MIT 许可证；大视觉编码器提供更丰富的视觉特征
- **适用场景**：当 Qwen2.5-VL-7B 描述质量不足时的替代方案

#### 备选 2：Qwen2.5-VL-72B / qwen-vl-max API

- **理由**：性能上限最高，GSC 论文直接使用此版本
- **适用场景**：质量上限评估、API 方式快速验证

#### 质量基线：GPT-4o

- **理由**：描述质量天花板，用于评估开源模型的质量差距
- **适用场景**：仅作为评估基线

#### 成本敏感场景：Gemini 2.0 Flash（或 2.5 Flash）

- **理由**：API 价格极低，适合大规模视频帧处理
- **适用场景**：大量视频帧的快速描述生成
