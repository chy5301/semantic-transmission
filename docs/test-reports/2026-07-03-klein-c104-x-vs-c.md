## 0. 目的

阶段 2 主报告在**单一视频**上得出「C@N12 质量最优、C>X 保真」。本补充在 **3 个不同 C104 场景**上只跑 X@N12 与 C@N12（负责人指定，不跑 N25、不跑 drop-in baseline），验证该规律是否**跨场景成立**。

| 项 | 值 |

| prompt | 每视频 Qwen2.5-VL 逐帧冻结一次，X/C 复用同一份；seed=0 |

## 3. 跨场景发现

1. **C 的保真优势跨场景稳健**：C@N12 在**全部 3 个场景**上 SSIM 更高、LPIPS 更低（093008 +0.051/−0.059、093113 +0.013/−0.024、121711 +0.012/−0.038）。恒挂真实关键帧持续把生成帧拉向真值 → 保真提升可复现，不是单场景偶然。
2. **X 纯帧间平滑度全部略优**：相邻帧 MAE / warp-error 三场景均 X 略低于 C，但差值很小（MAE 差 0.4–0.8），两者都远比阶段 1 drop-in（首视频 baseline 22）平滑。治闪烁主力仍是 prev 链。
3. **近静止稳定性场景相关**（关键新发现）：093008（近静止场景，原始窗口 MAE 1.53）**C 更稳**（6.34<7.69）；而 093113、121711（运动更多，原始窗口 MAE 2.39、5.18）**X 更稳**。印证设计 §9「静止关键帧 vs 区间末端运动」——场景运动大时，恒挂一张较旧的关键帧会与真实运动打架、反而略增抖动。
4. **121711 是难场景**：原始运动大（近静止窗口原始 MAE 5.18）、SSIM 仅 ~0.36、LPIPS ~0.65、warp ~14，两方案都更吃力；但 C 仍在保真上小胜。
5. **成本**：C（3 参考）比 X（2 参考）慢 ~28%（~23s vs ~18s/生成帧）、多 ~0.8GB 显存（20.6 vs 19.8GB），均在 24GB 内、无 OOM。

## 4. 结论

- **主报告结论跨场景成立**：参考帧补偿在不同 C104 场景下都产出低闪烁、且 C>X 的保真规律稳健。
- **配置建议微调**：
  - **画质优先 → C@N12**：保真跨场景稳健更优。
  - **速度优先或强运动场景 → X@N12**：快 ~28%、少显存，且在运动较多场景近静止稳定性反而略胜。
  - 「C 的近静止优势」并非普适——**近静止/低速场景选 C，强运动场景 X 已足够甚至更稳**。
- 与主报告一致，klein 关键帧主线 + 参考帧补偿方案成立，可进阶段 3 productionize。

## 5. 验收

- ✅ 3 C104 场景 × X@N12/C@N12 = 6 路真实 video→video（各 250 帧 896×672）跑通，失败 0、无 OOM、无分辨率回退。
- ✅ 裸 .h265 解码坑（无 fps 元数据）用 ffmpeg 转 25fps mp4 绕过。
- ✅ C 保真优势跨场景复现；近静止优势的场景依赖性被量化（补充主报告的单场景结论）。
