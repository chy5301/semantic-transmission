# 任务分析报告

## 项目概述

语义传输预研项目阶段一：调研与方案选型。本阶段的核心工作是信息收集和文档产出，不涉及代码开发。

**背景**：语义传输（Semantic Communication）是一个新兴研究方向，核心思路是"传输语义而非像素"——发送端用 AI 模型将视频压缩为文本描述+结构条件，接收端用生成模型还原，可实现极低码率（~0.01%）下的视频传输。

## 现状分析

### 已有资源
- ComfyUI 工作流：同事已构建 Z-Image-Turbo + ControlNet Union 工作流（图像→Canny边缘→文本+条件→图像重建）
- 模型组件：qwen_3_4b（text encoder）、z_image_turbo_bf16（diffusion）、ae（VAE）、Z-Image-Turbo-Fun-Controlnet-Union（controlnet）
- 本机未安装 ComfyUI

### 调研范围
1. **论文方向**：
   - 生成式视频语义通信（GVSC）——多模态语义融合+扩散模型
   - 生成式视频压缩（GVC，TeleAI）——0.01% 压缩率
   - 生成式语义编码（GSC）——超低码率场景
   - 扩散辅助极端视频压缩——轻量语义引导
   - 生成式多模态互增强视频语义通信（M3E-VSC）

2. **开源项目**：
   - OpenSemanticComm 框架
   - ComfyUI 生态中的文生视频/图生图工作流

3. **模型能力**：
   - 理解侧：Qwen-VL、InternVL、LLaVA 等多模态大模型
   - 生成侧：Stable Diffusion、Z-Image-Turbo、FLUX、Wan2.x、CogVideoX、HunyuanVideo
   - 条件控制：ControlNet（Canny、深度图、语义分割、姿态）

## 目标定义

### 预期结果
- 一份完整的调研报告，覆盖论文综述、开源项目评估、模型能力对比
- 明确的技术选型建议，指导阶段二的 ComfyUI API 原型搭建

### 成功标准
- 核心论文（≥5篇）已阅读并整理关键方法、指标和结论
- 开源项目可用性已评估
- 发送端（理解模型）和接收端（生成模型）各有 ≥2 个候选方案的对比
- ControlNet 条件类型的适用场景已明确
- 选型建议有充分的论据支撑
